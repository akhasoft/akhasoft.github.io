Class BaseCacheProvider

  • java.lang.Object
    • android.content.ContentProvider
      • akha.yakhont.BaseCacheProvider
  • All Implemented Interfaces:
    ComponentCallbacks, ComponentCallbacks2

    public class BaseCacheProvider
extends ContentProvider
    The ContentProvider which does not use predefined database schema but creates tables (or adds columns, if necessary) "on the fly".

    Some of available features are:

    • Checking for table or column existence
      • Checks if table exists: isExist(SQLiteDatabase, String)
      • Checks if column exists: isExist(SQLiteDatabase, String, String)
    • Database copying
      • Copy database: copyDb(Context)
      • Copy database: copyDb(Context, File, File)
    • SQL script(s) executing
      • Runs SQL script(s): execSql(Context, SQLiteDatabase, String)
      • Runs SQL script(s): execSql(SQLiteDatabase, String...)
    • Transactions support
      • Runs transaction: runTransactionBoolean(SQLiteDatabase, Callable)
      • Runs transaction: runTransaction(SQLiteDatabase, Callable)
      • Runs transaction: runTransaction(SQLiteDatabase, Runnable)
    • Database clearing
      • Clears database: clear(SQLiteDatabase)

    Example of declaration in the manifest file: 

     <manifest xmlns:android="http://schemas.android.com/apk/res/android"
     package="com.yourpackage" >

     <application ... >
     ...

     <provider
         android:authorities="com.yourpackage.provider"
         android:name="akha.yakhont.BaseCacheProvider"
         android:enabled="true"
         android:exported="false" />

     </application>
 </manifest>
    If you prefer to design another authority (for example, not based on the name of the package), please implement the Core.UriResolver interface and register it via the setUriResolver() method.
